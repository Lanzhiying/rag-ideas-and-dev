# MiniMax-M3 快速评估

> 2026-06-18，数据来源 HuggingFace API

---

## 概况

MiniMax-M3 是最新旗舰，M2.7 的下一代。一个 **VLM（视觉语言模型）**。

| 项目 | 值 |
|---|---|
| 模型 ID | `MiniMaxAI/MiniMax-M3` |
| 架构 | `minimax_m3_vl` |
| 总参数（BF16 safetensors） | **427B** |
| 文本层 | 60 层（前 3 稠密，57 MoE） |
| 每 token 激活 expert | 4 |
| 共享 expert | 1 |
| 上下文 | **1,048,576（1M）** |
| 视觉 | ✅ 原生多模态（CLIP 风格视觉编码器） |
| 许可证 | **other**（非标准开源）|

---

## 显存

| 版本 | 大小 | B200 192GB | 双 B200 384GB |
|---|---|---|---|
| BF16 原生 | 854 GB | ❌ | ❌ |
| MXFP8（官方）| ~410 GB | ❌ | ❌ |
| NVFP4（社区）| ~214 GB | ❌ | ✅ |
| 4-bit GGUF | ~107 GB | ✅ | ✅ |

B200 上只能跑 4-bit 量化版（~107GB），60GB 余量给 KV Cache。

## 为什么不选

1. **装不进好精度。** B200 上只能跑 4-bit 量化，427B 的模型 4-bit 质量损失大。
2. **许可证不明确。** "other" 不是标准开源协议，商用风险高。
3. **VLM 对当前场景不是加分项。** Qwen3.5-122B 也支持图文，不需要 M3 的视觉能力。
4. **没有中文评测数据。** 不知道在中文 RAG 场景下实际表现如何。

结论：M3 等了一年回头再看。当前阶段 Qwen3.5-122B 更实际。
