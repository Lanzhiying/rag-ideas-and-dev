# 2026-06-17: Embedding 模型太慢了怎么办

上了 RAGFlow，一切配好之后第一个瓶颈就是 Embedding。

本机跑 BGE 系列，一条文本好几秒。知识库要是有几千条文档，入库等到天荒地老。在线 API 倒是快——但花钱不说，数据还得传出去，跟当初想做本地化的初衷对着干。

能想到的路子：

1. 换轻量本地模型。牺牲点精度换速度，all-MiniLM、gte-small 这种
2. GPU 加速。~~本机没独显，此路不通~~
3. 加缓存。embed 过的文本别再重复算
4. 在线 API 过渡。先跑起来，找到最优解再切回本地
5. 混合方案。入库用在线（量大），查询用本地（实时、量小）

## 硬件情况

纯 CPU，16GB+ RAM。500MB 以下的模型比较从容，bge-large（~1.3GB）和 bge-m3（~2.2GB）能跑但会比较挣扎。

## 下一步

跑一轮速度测试。先把轻量和中等的几个模型都测一遍，看看实际差距有多大，再决定用哪个。
