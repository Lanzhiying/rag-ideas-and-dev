# 2026-06-17: Embedding 模型太慢了怎么办

上了 RAGFlow，一切配置好之后第一个瓶颈就是 Embedding。

本机跑 BGE 系列，一条文本几秒钟——知识库几千条文档的话，入库要等到天荒地老。在线 API 调用倒是快，但一是花钱，二是数据要传出去，跟「本地化」的初衷有冲突。

想到的可能方向：

1. **轻量本地模型** — 牺牲一点精度换速度（all-MiniLM、gte-small）
2. **GPU 加速** — 如果有独显，embedding 推理可以快几十倍
3. **缓存策略** — 对已经 embed 过的文本做缓存，避免重复计算
4. **在线 API 做过渡** — 先跑起来，找到最优方案后再切回本地
5. **混合方案** — 在线做入库（批量大量），本地做查询（实时少量）

需要先搞清楚：到底是模型大导致的慢，还是根本没用上 GPU。
