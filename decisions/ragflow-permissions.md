# RAGFlow 权限配置

2026-06-17 | 摸索中 | ~15 个子文件夹

---

## 实际场景

一个大文件夹下面有十几个子文件夹，不同的子文件夹对应不同的权限。大概是这样的结构：

```
big_folder/
├── 子文件夹A/
├── 子文件夹B/
├── ...
└── 子文件夹N/     ← 十几个
```

不是按内容类型分的，是按来源和用途。每个子文件夹天然就该有自己的可见范围。

---

## 现在倾向的方案

十几个知识库对于 RAGFlow 来说不算多。所以先不急着按权限等级合并，直接**一个子文件夹 → 一个知识库**。

理由：

- 权限最精确。每个知识库独立设团队，谁该看什么就是什么
- 语义最干净。不会把不相干的内容塞进同一个库互相干扰
- 维护简单。子文件夹增减就直接增减知识库，不会影响其他

唯一要解决的：跨知识库检索。用户的一个 query 可能需要同时搜好几个知识库。RAGFlow 如果支持这个功能就省事了。不支持的话，至少可以通过 API 自己拼——分别查几个库，合并排序返回。

---

## 操作步骤（计划）

1. 把每个子文件夹建一个对应的知识库
2. 搞清楚 RAGFlow 的团队设置能做什么——能不能按用户精确控制每个知识库的访问
3. 把用户分好组，每个知识库只邀请需要的人
4. 测一下 API key 的行为——会不会一个 key 能跨知识库访问不该访问的内容

---

## 待确认

- 跨知识库搜索——RAGFlow 有没有这个功能
- API key 的权限边界——绑用户还是绑知识库
- 知识库数量多了以后性能会不会下降
