# RAGFlow 权限配置

2026-06-17 | 刚开始摸索

---

## 场景

我一个人在搭，但最终可能要让别人用。至少有两种情况要考虑：

- **只有我自己**——所有权限全开，最简单
- **小团队共享**——同事可以查知识库，但不能乱改文档和配置

现在还不到生产部署的阶段，但权限这种事最好一开始就想清楚。后面再补会麻烦得多。

---

## RAGFlow 的权限到底有几层

翻了官方文档，信息比较散。目前理解大概是这么几层：

**系统级**

管理员 vs 普通用户。这个好理解，管理员能看所有知识库、管理用户。

**知识库级**

每个知识库有独立的「团队」设置。大概能控制谁能看、谁能改。具体的粒度还没摸清。

**API key 级**

RAGFlow 的 API key 怎么绑定的我还没搞明白。是跟用户走还是跟知识库走？如果一个用户有五个知识库，一个 key 能访问所有还是只能访问指定的？

---

## 需要搞清楚的事

- 知识库的三种可见性（「仅我」「团队」「所有人」）到底各能做什么
- 能不能让一个人只看某个知识库，看不到其他的
- API key 的权限范围是什么，能不能绑定到特定知识库
- 如果以后想接前端，权限要怎么传——总不能把 admin key 写在 js 里

---

## 怎么测

直接在 RAGFlow 里建两个测试知识库，开几个不同角色的账号，一个一个试：

1. 普通用户能不能看到没被邀请的知识库
2. API key 在不同知识库之间会不会串
3. 「团队」里的权限编辑到底能编辑什么

试完把结果记下来。文档没有就自己补。
