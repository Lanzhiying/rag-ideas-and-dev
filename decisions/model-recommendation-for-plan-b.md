# 方案B候选模型逐一评估（A100 80GB ×2, 160GB NVLink）

> 针对 RAG 场景（RAGFlow 本地部署），10-20 人并发，中文文档问答

---

## 一、为什么没推荐 Qwen3.6

Qwen3.6 目前只有一个模型：`nvidia/Qwen3.6-35B-A3B-NVFP4`（2026.05.27），不是 Qwen 官方发布。Qwen org 名下没有 Qwen3.6 的 Instruct 权重。

| | Qwen3.5-122B-A10B | Qwen3.6（现状） |
|---|---|---|
| 最大尺寸 | 122B MoE | 35B-A3B MoE |
| 发布方 | Qwen 官方 | NVIDIA 优化版 |
| 权重格式 | BF16/Q4 GGUF | NVFP4 |
| 用途 | 通用旗舰 | Agentic Coding 特化 |
> **2026-06-18 修正：** 原文档写 "FP8 ~65GB" 不准确。Qwen3.5-122B-A10B 原生是 BF16（~250GB）。官方有 FP8 版（`Qwen/Qwen3.5-122B-A10B-FP8`，~125GB）。65GB 对应的是 Q4 GGUF（4-bit 量化，~63GB）。方案 B（2×A100 160GB）实际跑的是 Q4 版，不是 FP8。

Qwen3.6-35B 太小了——35B MoE 跑 RAG 够用但没余量。等 Qwen 官方放出 Qwen3.6-122B 再上。**不是不推荐，是现在没有合适的尺寸。**

## 二、逐一评估

### DeepSeek V4-Flash

| | |
|---|---|
| 参数 | 158B MoE (FP8) |
| 许可 | MIT ✅ |
| Q4 显存 | ~87 GB |
| 发布日期 | 2026.04.22 |
| 下载量 | 222万 |

**优点：**
- MIT 许可，商业部署零限制
- GGUF 现成（antirez/deepseek-v4-gguf, 249万下载）
- SuperCLUE 中文综合 70.98 分，国内第一
- 社区最活跃，问题有人踩过

**缺点：**
- 激活参数未公开，KV Cache 不好预估
- thinking 模式 + tool_call 协议有坑（返回必须带 reasoning_content）
- AI 痕迹重（知网检测率 55-78%），输出需要 humanizer
- 才发布两个月，长期稳定性未知

**RAG 适用性**：★★★★☆
**推荐度**：备选。中文强但显存吃紧（87GB，余量 73GB），并发比 Qwen 少 8-10 人。

### Qwen3.5-122B-A10B

| | |
|---|---|
| 参数 | 122B 总/10B 激活 MoE |
| 许可 | Apache 2.0 ✅ |
| FP8/Q4 显存 | FP8 ~125GB / Q4 ~65GB |
| 发布日期 | 2026.02.24 |
| 下载量 | 76万 |

**优点：**
- 10B 激活参数明确，KV Cache 精确预估
- FP8 版仅 125GB（Q4 则 ~65GB），留 95GB 给并发 → 比 V4-Flash 多扛 30% 人
- 原生多模态（图+文+音+视），未来单模型覆盖更多场景
- 半年生态沉淀，社区文档/教程丰富
- 尺寸覆盖最全（0.8B~397B），同系 embedding/reranker 配套

**缺点：**
- 编程基准比 GLM/DS 低 5-7 分
- 长上下文 >64K 后质量明显下降
- Agent 工具调用偶有损坏的 function calls
- Apache 2.0 比 MIT 多一层专利条款（实际影响很小）

**RAG 适用性**：★★★★★
**推荐度**：主力。显存效率最高，生态最完整，坑最少。

### MiniMax M2.7

| | |
|---|---|
| 参数 | ~230B 总/~10B 激活 MoE |
| 许可 | other ⚠️ |
| Q4 显存 | ~127 GB |
| 发布日期 | 2026.04.09 |
| 下载量 | 213万 |

**优点：**
- 10B 激活极低，推理速度快（延迟比同级别模型低）
- 性价比极高（API $0.30/M tokens）
- AA 智能指数 61，开源最高
- M2 系列迭代三次（M2→M2.1→M2.5→M2.7），稳定性好

**缺点：**
- **许可证是 "other"，不是标准开源协议**。商用前必须确认条款
- Q4 需要 127GB，方案 B 的 160GB 跑完只剩 33GB 给 KV Cache，并发量最低
- Q4 精度下中文质量未充分验证（社区评测多为 API 版）
- 230B 总参数但 10B 激活——MoE 效率极高但总权重大，加载慢

**RAG 适用性**：★★★☆☆
**推荐度**：不推荐。许可证不明确是硬伤，显存效率也差于 Qwen 和 DS。

## 四、最终建议（2026-06-18 更新：硬件方案已改为 B200，见 decisions/hardware-b200-plan.md）

**前提：硬件已从 2×A100 160GB 改为单 B200 192GB（新加坡采购）。**

**主力：Qwen3.5-122B-A10B FP8（官方，~125GB）——B200 原生精度运行**

- B200 192GB → FP8 125GB 装上，余 67GB 给 KV Cache + 20 并发
- Apache 2.0 商用友好
- 半年生态成熟，RAGFlow 兼容验证
- 原生多模态（图文理解），后续可覆盖图片查询场景
- 同系 Qwen3-Embedding + Qwen3-Reranker 配套

**备选：DeepSeek V4-Flash Q4（~87GB）**

- 余量更充裕但 Q4 量化有质量损耗
- 中文质量略高于 Qwen，适合未来转向代码 Agent 场景

**未来关注：Qwen3.6-122B / Qwen3.7（如果 8-9 月发布）**

**不推荐：GLM-5.2（751B，硬件门槛 8×H100）、MiniMax-M3（427B，许可不明，装不进好精度）**

**一张卡跑主力，一张卡跑 embedding+reranker，预算余量全砸存储。**（注：最终改为 B200 单卡方案，详见 decisions/hardware-b200-plan.md）
