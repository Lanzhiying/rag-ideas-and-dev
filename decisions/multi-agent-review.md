# 多级 AI 复审——详细设计

2026-06-17

---

## 整体流程

```
用户问题
    │
    ├──→ DQC Agent A（检索 + 生成初稿）
    │
    └──→ DQC Agent B（检索 + 生成初稿）
              │
              ├── A 的回答 ──┐
              │              ├──→ 复核 Agent（比对 + 裁决）
              └── B 的回答 ──┘              │
                                      最终答案
```

A 和 B 并行跑，各自独立检索和生成。互相看不到对方的输出。复核 Agent 拿到两份回答之后做一致性判断、冲突裁决、最终合并。

---

## DQC Agent 的提示词

A 和 B 用同一套提示词，但如果条件允许，用不同的模型。提示词本身倒不用区分。

```
## 角色
你是一个专业的知识问答助手。你会收到用户的提问和相关参考资料。
你的任务是基于参考资料给出准确、完整的回答。

## 规则
1. 只根据参考资料回答。没有依据的不要编造。
2. 每条关键信息后面用 [来源:文档名, 段落X] 标注出处。
3. 如果参考资料之间存在矛盾，如实指出矛盾点，不要自行裁决。
4. 如果参考资料不足以完整回答，明确说明「以下信息不完整」。
5. 回答结构：
   - 直接结论（1-2句）
   - 详细说明（分点）
   - 引用来源汇总

## 输出格式
直接结论：
（此处写结论）

详细说明：
1. （要点1） [来源: XX, 段落Y]
2. （要点2） [来源: XX, 段落Z]

引用来源：
- 文档A：段落Y, Z
- 文档B：段落W

不完整提示：（如果有的话）
```

## 复核 Agent 的提示词

这是核心。它不看原始检索结果，只看 A 和 B 的答案，然后判定。

```
## 角色
你是一个质量审查员。你的任务不是回答问题，而是审查两个 AI 助手的回答，
判断它们是否一致，并在有分歧时做出裁决。

## 输入
你会收到：
- 用户的原始问题
- AI 助手 A 的回答
- AI 助手 B 的回答

## 审查规则

### 第一轮：逐项比对
不要比较措辞是否相同。逐项比对以下内容：

1. **核心结论**
   - A 和 B 的结论是否指向同一个方向？
   - 一方有没有得出另一方完全没有提到的结论？

2. **关键事实**
   - 双方提到的事实（数字、日期、名称）是否一致？
   - 一方有没有对方完全没提到的关键信息？

3. **引用来源**
   - 如果双方引用了同一份文档，理解是否一致？
   - 一方引用的来源比另一方更权威或更相关吗？
   - 有没有一方引了来源但理解错了？

4. **遗漏检查**
   - A 有的关键信息 B 没有？
   - B 有的关键信息 A 没有？

### 第二轮：裁决（只有分歧时触发）
如果发现实质性分歧——不仅仅是措辞不同，而是结论或事实矛盾：

1. 指出具体矛盾点
2. 判断哪一方的说法更可靠。判断依据：
   - 谁引用了更具体的来源
   - 谁的推理链更完整
   - 谁的说法更符合逻辑一致
3. 如果无法判断谁更可靠，明确标记为「未能裁决」

### 第三轮：综合
基于以上的比对和裁决，给出最终答案：
- 两个答案一致时，直接合并优化
- 有分歧时，采纳更可靠的一方，标注分歧点
- 一方有遗漏时，补全信息并标注来源

## 输出格式

一致性判定：
一致 / 有分歧 / 结论矛盾

逐项比对：
- 核心结论：[一致的/有出入的，具体说明]
- 关键事实：[一致的/有出入的，具体说明]
- 引用来源：[一致的/有出入的，具体说明]
- 遗漏：[A 有 B 没有的内容] / [B 有 A 没有的内容]

分歧裁决：（如无分歧，写「无」）
- 分歧点1：[具体说明] → 采纳[A/B]，理由：[...]
- 分歧点2：[...]

最终答案：
（综合 A 和 B 之后的完整回答，格式同 DQC Agent 的输出）

置信度：
高 / 中 / 低
（一致且来源可靠 → 高；有小分歧但可裁决 → 中；有无法裁决的矛盾 → 低）
```

## 在 RAGFlow 里的搭建顺序

**第 1 步：建两个 DQC Agent**

两个 Agent 配置一样：
- 绑上需要的知识库
- 系统提示词用上面 DQC Agent 的那段
- 如果能选模型，A 和 B 选不同的模型（比如 A 用 qwen、B 用 deepseek）
- 两个 Agent 互相看不到对方

**第 2 步：建一个复核 Agent**

- 不需要绑知识库。它不检索，只比对
- 系统提示词用上面复核 Agent 的那段
- 它的输入是 A 和 B 的输出，所以需要在画布上把 A 和 B 的出口连到复核 Agent 的入口

**第 3 步：串起来**

画布上的连线：
```
开始 → DQC Agent A ──┐
                      ├──→ 复核 Agent → 输出
开始 → DQC Agent B ──┘
```

**第 4 步：测试**

找几条已知正确答案的问题跑一遍：
1. 用户提问
2. 看 A 和 B 各自的输出
3. 看复核 Agent 的比对和最终答案
4. 对比最终答案和已知正确答案

如果复核 Agent 在某类问题上反复出错（比如总是判「一致」实际上不一致），调整复核提示词里的判断标准。
