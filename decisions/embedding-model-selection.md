# Embedding 模型选型考察（更新）

> 2026-06-30 | 硬件：E5 CPU × 8核，无 GPU，服务器待定  
> 数据：中英文混合，润滑油脂行业标准 + 技术文档  
> 背景：准备从 ZHIPU embedding-3（云端API）切到本地部署

## 核心约束

1. **CPU only**——目前服务器是 E5 × 8核，无 GPU，短期可能也加不了
2. **中英文混合**——文档不是纯中文，benchmark 要看多语综合分，不能只看中文
3. **专业领域**——润滑油脂术语密集，对 embedding 的训练数据覆盖有要求
4. **数据安全**——配方数据不能外传，必须本地，不能用云端 API（包括 ZHIPU、OpenAI、Cohere）

## 选型结论

### 当前硬件（E5 CPU，无 GPU）

**首选：Qwen3-Embedding-0.6B**
理由：
- CPU 可跑，速度好（一份文档几十秒）
- 多语综合分 64.33，中英混合场景靠谱
- Apache 2.0，无协议风险
- 中文检索 72.22，对一般问答够用

**备选：BGE-M3**
理由：
- 中文检索分更高（79.11 vs 72.22），如果发现 0.6B 不够用可以切
- 但多语综合分低（59.56），英文容易被拖后腿

**临时方案：jina-embeddings-v4**
- MMTEB 66.49，比 Qwen3-0.6B 高 2 分
- 长文档支持好（67.11），对标准 PDF 可能有优势
- 但数据不足，建议放最后试

### 未来加 GPU 后

直接上 **Qwen3-Embedding-8B**（70.58）或 **QZhou-Embedding-Zh**（CMTEB 第一）。
两者都是 7~8B 级别，一张 A100 / 4090 就能跑。

## 已知不足

- QZhou 只有 7B/8B 版本，没有小模型
- 没有在润滑油脂专业数据集上做过对比测试——benchmark 分数只能参考，实际效果需要在自己数据上验证
- 建议先切到 Qwen3-0.6B 跑一段时间，如果召回率不满意再换
