# Embedding 模型选型考察

> 状态：进行中
> 日期：2026-06-17
> 背景：RAGFlow 本地部署，需为中文知识库选 Embedding 模型

---

## 问题描述

1. **本地跑 Embedding 太慢** — 自己的机器推理速度跟不上实际使用需求
2. **在线模型参差不齐** — 不同模型在中文上的表现差异大，没有统一的评估标准
3. **缺乏系统性对比** — 需要一个可量化的方法来判断哪个模型适合我的场景

---

## 核心需求

- **语言**：中文为主（可能包含英文文档）
- **场景**：RAG 检索，关注召回率（能不能找到对的 chunks）
- **部署**：优先本地，可接受在线 API 作为过渡方案
- **速度**：需要在实际可用的响应时间内完成 embedding

## 评估维度（待细化）

- [ ] **检索精度** — 在某测试集上的 Recall@K、MRR
- [ ] **推理速度** — 本地模型的 tokens/sec（GPU/CPU）
- [ ] **API 延迟** — 在线模型的平均响应时间
- [ ] **内存占用** — 本地模型显存/内存需求
- [ ] **中文理解** — 对中文语义、近义词的区分能力
- [ ] **长文本支持** — 能否处理超过 512 token 的 chunk
