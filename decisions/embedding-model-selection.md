# Embedding 模型选型考察

> 状态：进行中
> 日期：2026-06-17
> 约束：**纯 CPU，16GB+ 内存，无 GPU**

---

## 问题描述

1. **本地跑 Embedding 太慢** — CPU 推理速度是主要瓶颈
2. **在线模型参差不齐** — 不同模型在中文上的表现差异大
3. **缺乏系统性对比** — 需要量化数据支撑选型

---

## 硬件约束分析

| 资源 | 现状 | 影响 |
|------|------|------|
| GPU | ❌ 无 | CPU 推理，速度慢 10-50x |
| 内存 | ✅ 16GB+ | 可以跑 1GB 级别的模型 |
| 存储 | ✅ 充足 | 无限制 |

**结论**：优先轻量模型（<500MB），大模型（bge-large ≈1.3GB, bge-m3 ≈2.2GB）仅作精度参考。

## 评估维度

- [ ] **推理速度** — texts/sec，这是核心指标
- [ ] **内存占用** — 模型加载后的 RAM 占用
- [ ] **中文检索精度** — Recall@K（需要测试集）
- [ ] **加载时间** — 冷启动到可用的时间

---

## 预期策略

```
阶段一：用小模型（bge-small / gte-small）扛住速度
阶段二：如果精度不够，换 bge-base
阶段三：极端精度需求时，考虑在线 API 兜底
```
